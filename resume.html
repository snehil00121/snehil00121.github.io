Engagement Overview

This document outlines the methodology, tooling, and deliverables I apply during web application vulnerability assessments. It is provided as a reference for the proposed engagement on complymarket.cloud (Azure Kubernetes Service).

Methodology

Phase 1 — Reconnaissance

  • Passive fingerprinting: HTTP headers, TLS configuration, server banners
  • Subdomain and endpoint enumeration via Amass, Subfinder, and ffuf
  • JavaScript file analysis for exposed API keys, endpoints, and secrets
  • External AKS surface review: publicly exposed services, ingress rules, and misconfigured endpoints

Phase 2 — Vulnerability Assessment

  • Manual web application testing across the full OWASP Top 10
  • Authentication and session management analysis
  • Broken access control and IDOR testing
  • SQL Injection and XSS — manual verification, not scanner-only
  • API endpoint enumeration and security misconfiguration review
  • Container and Kubernetes misconfiguration review using Trivy (external surface only, as scoped)
  • Automated surface coverage via Nuclei and Burp Suite Pro active scan

Phase 3 — Reporting

  • All findings rated by CVSS v3.1 severity: Critical / High / Medium / Low
  • Each finding includes: description, proof of concept, business impact, remediation steps
  • Report written for both technical and non-technical stakeholders
  • Delivered as a clean PDF within the agreed timeline

Tooling

  • Web Application: Burp Suite Professional, OWASP ZAP
  • Recon & Enumeration: Amass, Subfinder, ffuf, Gobuster, Shodan
  • Scanning: Nuclei, Nmap, Nikto, Nessus
  • Exploitation: Metasploit, SQLMap (manual verification always applied)
  • Cloud: Trivy (container/AKS misconfiguration), Cloudsploit
  • Scripting: Python, Bash for custom automation
